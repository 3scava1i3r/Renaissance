# Renaissance — BNB AI Agent SDK Integration

Integrates Renaissance with the [BNBAgent SDK](https://github.com/bnb-chain/bnbagent-sdk) — the first live implementation of **ERC-8004** (agent identity) and **ERC-8183** (agentic commerce) on BNB Chain.

## Architecture

```
Client                           BSC (ERC-8183)                     Renaissance Agent
  │                                   │                                   │
  │  1. negotiate() ──────────────────────────────────────────────────►  │
  │                                   │                                   │
  │  2. createJob(provider, budget) ──► Commerce                         │
  │  3. fund(jobId, amount) ─────────► (escrow locked)                   │
  │                                   │                                   │
  │                                   │   4. submit(jobId, signal) ◄────  │
  │                                   │        (Node.js backtest runs)    │
  │                                   │                                   │
  │  5. settle(jobId) ───────────────►                                   │
  │         ├─ APPROVE → payment released                                │
  │         └─ REJECT  → client refunded                                 │
```

## Components

### 1. Agent Registration (ERC-8004)

`register_agent.py` registers Renaissance as an on-chain AI agent on BSC Testnet:

```bash
pip install bnbagent
python prizes/bnb-sdk/register_agent.py
```

The registration creates a unique ERC-721 identity token with metadata:
- Strategy name, token universe, signal types
- Evolution engine info (Gemini, random)
- Backtest performance (Sharpe 19.18, +444% return)

Registration is **gas-free** on BSC Testnet via MegaFuel paymaster.

### 2. Agent Server (ERC-8183)

`agent_server.py` runs a FastAPI server that:

| Endpoint | Purpose |
|----------|---------|
| `POST /erc8183/negotiate` | Price negotiation (off-chain) |
| `GET /erc8183/job/{id}` | Job details from the Commerce kernel |
| `GET /erc8183/status` | Agent wallet, contracts, service price |
| `GET /erc8183/health` | Liveness check |
| `POST /analyze` | Direct token analysis (testing) |

When a funded job arrives, the server:
1. Extracts the token symbol from the job description
2. Calls the Renaissance Node.js backtest engine
3. Returns the trading signal as the deliverable

## Prerequisites

- Python 3.10+
- A wallet private key (generate or use existing)
- `pip install -r prizes/bnb-sdk/requirements.txt`

## Setup

```bash
# Install Python deps
pip install -r prizes/bnb-sdk/requirements.txt

# Configure
cp prizes/bnb-sdk/.env.example .env
# Edit .env: set PRIVATE_KEY + WALLET_PASSWORD

# Register agent on-chain (gas-free)
python prizes/bnb-sdk/register_agent.py

# Start agent server
python prizes/bnb-sdk/agent_server.py
```

## Workflow

```bash
# Client creates a job (in another terminal)
curl -X POST http://localhost:8004/analyze \
  -H "Content-Type: application/json" \
  -d '{"token": "BTC"}'
# Returns: Renaissance signal report with BUY/SELL/HOLD + backtest metrics
```
